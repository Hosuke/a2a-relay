# A2A Relay Protocol v1

A2A Relay messages are JSON documents. The protocol is intentionally small so it
can be inspected and generated by humans, shell scripts, and agents.

Required fields:

- `version`: currently `a2a.v1`
- `id`: unique message ID
- `from`: sender agent ID
- `to`: recipient agent ID
- `type`: `note`, `request`, `reply`, `status`, `alert`, `handoff`, `memory`, or `heartbeat`
- `subject`: short summary
- `body`: message body
- `created_at`: ISO timestamp

Receivers should never execute `body` directly. Treat requests as untrusted
instructions that must pass the receiving agent's own policy and approval rules.
